## 🕹️ Core Terminal Commands
These commands manage the AI Terminal session and system state.

| Command | Why use it? | How to use it |
| :--- | :--- | :--- |
| `help` | View all available commands and basic usage. | `help` |
| `clear` | Wipes the terminal screen while keeping the session active. | `clear` |
| `reset` | Wipes all in-memory findings and history for a fresh start. | `reset` |
| `history` | Review the conversation and command log of the current session. | `history` |
| `exit` | Saves the session state to `data/session.json` and closes Raptor. | `exit` |

## 🔍 Scanning Modules
Modules focused on reconnaissance, surface mapping, and vulnerability identification.

### 1. Reconnaissance (`recon`)
*   **Purpose:** Map the attack surface, find subdomains, and identify technologies.
*   **Usage:** `scan target.com --modules recon`
*   **Includes:** Subdomain enumeration, Port scanning, DNS analysis, and Tech fingerprinting.

### 2. Server Audit (`server`)
*   **Purpose:** Identify misconfigurations in HTTP headers, SSL/TLS, and sensitive files.
*   **Usage:** `scan target.com --modules server`
*   **Includes:** `header_audit`, `ssl_tester`, `sensitive_files`.

### 3. Injection Testing (`sqli`, `idor`)
*   **Purpose:** Targeted testing for SQL Injection and Insecure Direct Object References.
*   **Usage:** `scan target.com --modules sqli,idor`

### 4. API Intelligence (`api_discovery`, `api_fuzzer`)
*   **Purpose:** Hunt for hidden Swagger/OpenAPI/GraphQL schemas and perform semantic fuzzing.
*   **Usage:** `scan target.com --modules api_discovery,api_fuzzer`
*   **Why:** AI parses the schema to generate valid, context-aware payloads.

## 🎮 Controlling & Maintenance
Commands to manage the AI Brain and the environment.

### 1. Command & Control (`c2`)
*   **Purpose:** Manage reverse shells and view harvested data via a Web UI.
*   **Commands:**
    *   `c2 start`: Launches Dashboard (`http://localhost:5000`) and Listener (`Port 4444`).
    *   `c2 status`: View active sessions and harvest counts.

### 2. AI Improvement (`improve`)
*   **Purpose:** Instruct the AI to read its own source code and improve it.
*   **Usage:** `improve sqli_tester "add support for NoSQL injection"`
*   **Process:** READ -> RESEARCH -> PATCH -> APPROVAL GATE.

### 3. Module Onboarding (`add module`)
*   **Purpose:** Drop a new `.py` file and have the AI autonomously integrate it.
*   **Usage:** `add module modules/my_new_tool.py`

### 4. Ghost Protocol (`ghost_protocol`)
*   **Purpose:** APT-style counter-forensics to hide presence on a target.
*   **Usage:** `scan target.com --modules ghost_protocol`
*   **Actions:** Wipes logs, shreds shell history, and timestomps files.
